# Skill: Write Plan
Create detailed implementation plans.

## Auto-Trigger
**When:** "plan", "how should I", "approach", "strategy", "roadmap"

## Plan Template

### 1. Objective
```markdown
**Goal:** [One sentence describing end state]
**Success Criteria:**
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3
```

### 2. Current State Analysis
```markdown
**What exists:**
- [existing component 1]
- [existing component 2]

**What's missing:**
- [gap 1]
- [gap 2]

**Constraints:**
- [constraint 1]
- [constraint 2]
```

### 3. Approach Options
```markdown
**Option A: [Name]**
- Pros: [list]
- Cons: [list]
- Effort: [S/M/L]
- Risk: [Low/Medium/High]

**Option B: [Name]**
- Pros: [list]
- Cons: [list]
- Effort: [S/M/L]
- Risk: [Low/Medium/High]

**Recommendation:** Option [X] because [reason]
```

### 4. Implementation Steps
```markdown
**Phase 1: Foundation** (Day 1-2)
- [ ] Step 1.1: [description]
- [ ] Step 1.2: [description]
- Milestone: [deliverable]

**Phase 2: Core** (Day 3-5)
- [ ] Step 2.1: [description]
- [ ] Step 2.2: [description]
- Milestone: [deliverable]

**Phase 3: Polish** (Day 6-7)
- [ ] Step 3.1: [description]
- [ ] Step 3.2: [description]
- Milestone: [deliverable]
```

### 5. Risks & Mitigations
```markdown
| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| [risk 1] | Medium | High | [plan] |
| [risk 2] | Low | Medium | [plan] |
```

### 6. Resources Needed
```markdown
**Technical:**
- [tool/service 1]
- [tool/service 2]

**Knowledge:**
- [docs to read]
- [skills needed]

**Dependencies:**
- [external dependency 1]
- [team dependency 1]
```

## Quick Planning Questions
```
1. What does "done" look like?
2. What's the smallest useful increment?
3. What could go wrong?
4. What do I not know yet?
5. Who else needs to be involved?
```
