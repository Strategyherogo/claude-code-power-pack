Create conference talk proposals and slides.

## CFP (Call for Papers) Proposal

### Template
```markdown
## Talk Title
[Specific, benefit-focused, intriguing]

## Abstract (300 words max)
[Paragraph 1: Hook - why this matters now]

[Paragraph 2: What attendees will learn]

[Paragraph 3: Key takeaways]

## Outline
1. Introduction (5 min)
2. [Section 1] (10 min)
3. [Section 2] (10 min)
4. [Section 3] (10 min)
5. Q&A (5 min)

## Key Takeaways
- Attendees will learn [specific skill/knowledge]
- Attendees will be able to [specific action]
- Attendees will understand [specific concept]

## Target Audience
- [Level]: Beginner / Intermediate / Advanced
- [Role]: Developers / Designers / Product / etc.
- [Prerequisites]: None / Familiarity with X

## Bio (100 words)
[Name] is [role] at [company]. They [relevant experience]. [Notable achievement]. [Personal touch]. Find them at [@handle] or [website].

## Previous Speaking (if any)
- [Conference] - [Talk title] - [Link to video if available]
- [Conference] - [Talk title]
```

## Talk Structure (30 min)

### Opening (3 min)
```
Hook: Story, question, or surprising fact
Credibility: Brief intro (30 sec max)
Promise: What they'll learn
Roadmap: Brief outline
```

### Body (22 min)
```
Section 1: Foundation (7 min)
- The problem/context
- Why it matters
- Key concept

Section 2: Deep Dive (8 min)
- The solution/technique
- Live demo or walkthrough
- Examples

Section 3: Application (7 min)
- How to apply this
- Common pitfalls
- Real-world examples
```

### Close (5 min)
```
Summary: Key takeaways
Action: What to do next
Resources: Where to learn more
Q&A setup
```

## Slide Design Guidelines
```
Text:
- Max 6 words per line
- Max 6 lines per slide
- 30+ pt font size

Visuals:
- One idea per slide
- Full-bleed images
- Minimal bullet points
- Code in 24+ pt font

Flow:
- Title slide
- Agenda slide
- Section dividers
- Takeaway slides
- Resource slide
- Thank you + contact
```

## Speaking Tips
```
Preparation:
□ Practice out loud 3x minimum
□ Time each section
□ Prepare for tech failures
□ Know your first 30 seconds cold

Delivery:
□ Arrive early, test tech
□ Make eye contact
□ Pause for effect
□ Repeat questions from audience
□ Stay within time

After:
□ Share slides
□ Thank organizers
□ Connect with attendees
□ Gather feedback
```

## Common Talk Types
```
Tutorial: Teach a skill step-by-step
Case Study: Share real project learnings
State of X: Overview of topic/technology
Hot Take: Contrarian perspective
Live Coding: Build something on stage
Panel: Facilitate discussion
Lightning: 5-10 min focused talk
```
